Todo List

Student Choice Survey

For this project, your performance assessment is to make a website similar to the MiniBlog using Materialize, Handlebars, and Backendless.

You are going to be supplied with two sets of notes, one for Handlebars and one for Backendless. These explain the same concepts covered in the videos but in more detail.

Handlebars: https://docs.google.com/document/d/1S2si4PCoyxICSrWM6KGe2p7BiUBOBvuPQbukanLlEwI/edit?usp=sharing

Performance Assessment Difficulty Tiers

Basic

These milestones are designed to see if you can implement a basic data-driven website with minimal guidance.

  • AS ALWAYS, each milestone needs to be pushed separately to a new Bitbucket Repository.

Milestone 1: Design a home page

  • Create the basic layout of your page using Materialize. THE DESIGN MUST BE DIFFERENT FROM THE MINIBLOG.

Milestone 2: Allow anyone to add a todo task

  • Add a basic form on the top of the page that will insert data into a backendless table.

Milestone 3: Render all your tasks on your page

  • Using handlebars, render all of the tasks in the todo table.

Milestone 4: Show the date the task was created

  • Show the date in the task

Intermediate (B level work)

These milestones will require a deeper understanding of what we have done using backendless and handlebars.

Milestone 1: Allow your tasks to be marked as complete

  • Allow your tasks to be marked as complete, and have them be whited out (but not deleted), and allow them to be unmarked back to incomplete.
  • Hints: use the handlebars "{{#if }}" and "{{#else}}" expressions to render different templates based on if a task is completed or not. http://handlebarsjs.com/builtin_helpers.html

Also use a jQuery event to mark tasks as completed an incomplete in backendless.

Milestone 2: Allow your tasks to be deleted

  • Task must be deleted from backendless, and should not be rendered on the page.

Milestone 3: Add a persistent login to your website

  • Create a login page, and make sure the login form doesnt render again on page refresh.

Milestone 4: Only let the user see tasks that they created

  • Search your tasks table for tasks that you created and only render them.
    var taskStorage = Backendless.Persistence.of( Tasks);
    var dataQuery = {
         condition: "author = " + YOUR_ID_HERE
    };
    var myTasks = taskStorage.find( dataQuery );

Milestone 5: Don't allow empty task lists

  • If your form is empty use toast to notfiy the user and don't allow them to insert.

Advanced (A level work)

This is all work we have not covered in videos yet, but if you can figure them out I will award extra credit.

Milestone 1: Host your website

  • https://pages.github.com/

Milestone 2: Only show tasks form to logged in users

  • Use handlebars to check if a user is logged in.

Milestone 3: Allow your users to add multiple task lists

  • Think something similar to https://trello.com/
